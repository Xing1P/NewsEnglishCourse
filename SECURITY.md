# Security Policy

## Supported Versions

Security fixes are considered for the current `main` branch and the latest published release.

## Reporting A Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Email security reports to `phanpiseth9@gmail.com` with:

- A clear description of the issue.
- Steps to reproduce or a proof of concept, if available.
- The affected version, operating system, and environment.
- Any suggested mitigation.

The maintainer will review reports as soon as practical. If the issue is confirmed, fixes and disclosure timing will be coordinated before public details are shared.

## Data And Privacy Notes

News English Course stores its course library locally in SQLite and uses the locally installed Gemini CLI for generation. Do not attach private learner data, private articles, credentials, or local database files to public issues.
